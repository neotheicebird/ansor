Here I am collecting awesome ansors (read: answers) found in SO to great (mostly python related) questions. Here they are:

1 Late binding 

fns = []
for i in range(5):
    def fn(x):
        return x + i
    fns.append(fn)

# Passing a 1 to each function in the list I would expect an output of [1, 2, 3, 4, 5].
print [f(1) for f in fns]    # prints [5, 5, 5, 5, 5]

The point to understand here is that python uses a late binding approach and the function call takes place at runtime. So, Murphy's law. This is how it is fixed:

for i in range(5):
    def fn(x, i=i):  # <-----
        return x + i
    fns.append(fn)

print [f(1) for f in fns]    # prints [1, 2, 3, 4, 5]

Why is this working this way? The answer is in the closure of the environment they're defined in. That is scope of a variable or which namespace python would search for the variable when it is executed. So in the first case i is global, but in the second one by using a default i the value of i is made local to the function. more so

Is there a more pythonic way to do this. There sure are many:

fns = []

for i in range(5):
    def fn(j):
        def inner(x): 
            return x + j
        return inner
    fns.append(fn(i))

print [f(1) for f in fns]    # prints [1, 2, 3, 4, 5]

# or

from functools import partial

fns = []
def fn(x, i):
    return x+i
for i in range(5):\
    fns.append(partial(fn, i))

print [f(1) for f in fns]    # prints [1, 2, 3, 4, 5]
